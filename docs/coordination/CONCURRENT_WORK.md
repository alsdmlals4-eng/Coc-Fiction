# Coc-Fiction 동시 작업 규칙

이 저장소는 여러 채팅·작업자가 동시에 수정할 수 있다. `main`은 통합 기준으로만 사용하고 실제 작업은 독립 브랜치에서 수행한다.

## 필수 규칙

1. 작업 시작 직전에 `main`의 최신 커밋과 열린 PR을 확인한다.
2. 작업마다 고유 브랜치를 사용한다. 예: `ops/...`, `fiction/...`, `revision/...`.
3. 다른 작업자가 수정 중일 가능성이 있는 원고·설정·루트 진입 파일은 필요 없이 덮어쓰지 않는다.
4. 기존 파일을 변경할 때 책임 원본과 untouched 소비자를 먼저 확인한다.
5. force push·hard reset·무검토 덮어쓰기를 금지한다.
6. PR 직전에 최신 `main`과의 차이, 겹치는 파일, 깨진 참조, 누락된 소비자를 다시 확인한다.
7. 충돌 가능성이 있으면 자동 병합보다 `KEEP / MERGE / UPDATE / DEFER` 판정을 먼저 기록한다.
8. 실행하지 않은 검사와 해결하지 않은 충돌은 `PASS`로 표시하지 않는다.

## 현재 책임 분리

- 소설 운영 Skill·템플릿·검증 구조: `main`에 병합됨.
- 현행 작품 코어·Canon·원고·인수인계: `fiction/` 경로가 책임진다.
- Google Docs는 사람이 읽는 통합 뷰이며, GitHub의 세분화된 책임 원본을 대체하지 않는다.
- 과거 부트스트랩 기반 이전 PR은 실패한 생성 실험으로 취급하며 현행 작업 입력에서 제외한다.

## 새 작업 시작 전 읽기

```text
[소설]/00_운영체계/START_HERE.md
→ fiction/FICTION_MASTER.md
→ fiction/ACTIVE_CONTEXT.md
→ fiction/CANON_REGISTRY.json
→ 현재 5화 원고 묶음
```

이 문서는 동시 작업 안전 규칙만 책임지며 소설 내용·설정·문체 정본이 아니다.
