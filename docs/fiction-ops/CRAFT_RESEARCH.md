# 소설 작법·퇴고 Research Notes

이 문서는 외부 작법을 작품에 강제하는 규칙집이 아니다. 현재 문제를 진단할 때 필요한 렌즈와 출처를 제공하며, 적용 여부는 작품 코어·장르·문체·사용자 승인으로 결정한다.

## 1. 인물·플롯 연결

Purdue OWL의 fiction 자료는 인물의 욕망·목표와 장애를 플롯에 연결하고, 배경·신념·결함·행동의 일관성으로 개연성을 만든다고 설명한다.

적용:

- 인물마다 장기 목표와 장면 목표를 구분한다.
- 목표가 사건을 발생시키는지, 사건이 인물 선택을 강제하는지 확인한다.
- 설정표의 정보 전부를 설명하지 않고 행동·선택·물리적 세부로 드러낸다.
- 결함은 임의 장식이 아니라 배경·신념·경험에서 파생되게 한다.

출처:

- https://owl.purdue.edu/owl/subject_specific_writing/creative_writing/writers/fiction-basics/characters_and_fiction_writing.html
- https://owl.purdue.edu/owl/subject_specific_writing/creative_writing/writers/fiction-basics/writing_compelling_characters.html
- https://owl.purdue.edu/owl/subject_specific_writing/creative_writing/writers/fiction-basics/building_and_revealing_characters.html

## 2. 점진 확장형 플롯 설계

Randy Ingermanson의 Snowflake Method는 이야기의 작은 핵에서 시작해 한 문장·단락·인물·장면 목록으로 점진 확장하는 방식이다. 장편 복잡도 관리에 유용하지만 모든 작가에게 맞는 공식은 아니다.

적용:

```text
한 문장 전제
→ 주요 전환을 포함한 짧은 시놉시스
→ 인물별 욕망·갈등·변화
→ 인과 플롯
→ 장면 목록
```

초고가 이미 있다면 이 순서를 거꾸로 적용해 현재 작품을 압축하고 구조 결함을 찾는다.

출처:

- https://www.advancedfictionwriting.com/articles/snowflake-method/

## 3. 장면 진단

Ingermanson이 정리한 Scene/Sequel 렌즈는 능동 장면을 `목표→갈등→좌절/전환`, 반응 장면을 `반응→딜레마→결정`으로 본다. 이는 장면을 판정하는 **선택적 진단 도구**이며 모든 장면을 동일한 공식으로 강제하지 않는다.

Motivation-Reaction Unit은 외부 자극과 POV 인물의 내적·행동적 반응 순서를 점검하는 미시 렌즈다. 반응의 생리·감정·행동 순서가 뒤집혀 독자 체감이 끊기는지 확인하는 데 쓴다.

출처:

- https://www.advancedfictionwriting.com/articles/writing-the-perfect-scene/
- https://www.advancedfictionwriting.com/blog/2017/05/14/start-end-scene/

주의:

- 배경·분위기·여운이 중요한 장르에서는 장면의 변화가 사건이 아니라 인식·관계·정서일 수 있다.
- 이 렌즈가 작품의 리듬과 미학을 훼손하면 `REJECT`하거나 부분 적용한다.

## 4. 인과와 인물 의도

Narrative planning 연구는 사건의 논리적 인과와 인물 행동을 설명하는 목표·의도가 함께 있어야 이야기 이해도가 높아진다고 본다.

적용:

- 중요한 사건마다 `원인 / 선택 주체 / 의도 / 대가 / 결과 / 다음 압력`을 기록한다.
- 플롯에 필요한 행동이 인물에게 필요한 이유도 있는지 확인한다.
- 우연이 문제를 만들 수는 있어도 핵심 해결을 대신하지 않게 한다.

출처:

- https://arxiv.org/abs/1401.3841

## 5. 역개요

UW–Madison Writing Center의 reverse outline은 완성된 초안에서 각 단락의 핵심 기능을 추출해 구조·누락·순서 문제를 확인하는 방법이다. 소설에서는 단락보다 장면 단위로 확장한다.

소설 역개요 필드:

```text
장면 ID
POV·시간·장소
목표 또는 초점
핵심 갈등·발견
전환·결과
새 정보
인물·세계 상태 변화
다음 장면에 주는 압력
분량
```

출처:

- https://writing.wisc.edu/handbook/reverseoutlines/
- https://writing.wisc.edu/handbook/revisinglongprojects/

## 6. 퇴고와 교정 분리

UNC Writing Center는 revision, editing, proofreading을 서로 다른 단계로 구분한다. 먼저 목적·내용·전체 구조를 다시 보고, 이후 문장과 표현을 편집하며, 표면 오류 교정은 마지막에 수행한다.

Coc-Fiction 적용 순서:

```text
작품 코어·전체 구조
→ 인과·장면 순서
→ 연속성·POV·정보
→ 문장·리듬·대화·묘사
→ 맞춤법·띄어쓰기·문장 부호
→ 최종 proofread
```

한 패스에서 모든 오류를 동시에 찾지 않고 2~3개 의제에 집중한다. 구조 수정 뒤에는 하위 교정을 다시 확인한다.

출처:

- https://writingcenter.unc.edu/tips-and-tools/revising-drafts/
- https://writingcenter.unc.edu/tips-and-tools/editing-and-proofreading/

## 7. 한국어 교정

맞춤법·띄어쓰기·문장 부호는 개인 취향이 아니라 국립국어원의 현행 어문 규범과 표준국어대사전을 우선 확인한다. 다만 소설의 의도적 구어·방언·파편문·리듬은 오류와 연출을 구분해 기록한다.

출처:

- https://korean.go.kr/kornorms/main/main.do
- https://stdict.korean.go.kr/

## 8. 참고문장·연출 사용 원칙

참고 자료를 저장할 때 문장을 수집하는 데서 끝내지 않는다.

```text
출처·권리 상태
→ 짧은 발췌
→ 문장의 기능
→ 사용한 장치
→ 독자에게 생기는 효과
→ 전이 가능한 추상 원리
→ 현재 작품에서 바꿔야 할 요소
→ 모사·복제하면 안 되는 고유 요소
```

- 사용자 제공·퍼블릭 도메인·허용된 자료를 우선한다.
- 저작권 있는 자료는 비평에 필요한 짧은 발췌만 사용한다.
- 특정 현존 작가의 고유 문체를 재현하는 프롬프트를 만들지 않는다.
- 단어·비유·문장 구조를 복사하지 않고 기능을 다른 소재·리듬·시점으로 재설계한다.
- 외부 작품의 플롯을 이름만 바꿔 이식하지 않는다.

상세 기록 형식은 `REFERENCE_CARD_TEMPLATE.md`를 사용한다.
